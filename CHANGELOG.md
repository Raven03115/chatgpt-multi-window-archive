# Changelog

本文件記錄各版本的重要功能、修正與行為變更。

## v4.6.0 — 2026-07-17

- 新增活動窗格頂部中央對話提示。
- 切換窗格時顯示目前窗格編號及對話或頁面名稱。
- 提示約 3 秒後自動淡出，不阻擋輸入、滑鼠、捲動、按鈕或文字選取。
- 支援一般對話、Project 對話、新對話與其他可辨識頁面。
- 新對話取得正式標題後可顯示更新後名稱。
- Settings、Search、記憶管理及其他 overlay 顯示時不顯示提示。
- 快速切換窗格時不堆疊、不殘留舊提示。
- 完整自動測試增加至 110 項，兩個 Electron fixtures 通過。
- 保持記憶管理、搜尋、Settings、排程、路由與既有窗格行為。
- 4 格或 6 格布局切換對話時可能短暫顯示背景，仍為已知限制。

## v4.5.9 — 2026-07-16

- 修正 Settings 記憶管理新增／更新輸入框無法輸入的問題。
- 修正記憶管理介面的點擊被誤判為 Settings 背景並自動關閉。
- 修正記憶管理子視窗 X／Escape 錯誤關閉整個 Settings。
- 修正關閉 Settings 後左側欄殘留 blur／backdrop 的問題。
- 修正連續開關記憶管理時交替失效的問題。
- 保持風格與語氣選單、搜尋、窗格輸入、Project 與既有路由行為。
- 完整自動測試增加至 74 項，Electron fixture 通過。
- 4／6 格切換對話時可能短暫顯示背景，仍為已知限制。

## v4.5.8 — 2026-07-13

- 修正排程頁在 Electron 中只顯示範本、無法顯示 active tasks 的問題。
- 僅針對 ChatGPT automations listing GET XHR 移除 Electron User-Agent token，其他頁面與請求維持原始 User-Agent。
- 修正 compact confirmation dialog 偵測與 Upgrade menu precedence。
- 修正 Settings 右側 conversation 區域點擊穿透到底層 pane。
- 修正 Settings 透明 root 沒有官方 backdrop 時，右側點擊無法關閉設定。
- 修正 Upgrade 關閉 timer 覆蓋後續 Settings shape 的競態。
- 修正 Upgrade 關閉手勢因 DOM 移除而重新命中底層 Upgrade menu item，造成畫面閃回。
- 完整自動測試增加至 68 項，Electron fixture 通過。
- 保持一般對話、Project、Search、刪除確認框、pane 與既有功能。
- 4 格或 6 格切換對話的短暫背景閃爍仍為已知限制，本版未處理。

## v4.5.7 — 2026-07-12

- 修正點擊一般對話與 Project 對話的三點操作按鈕時，可能誤切換或開啟對話的問題。
- 修正三點操作選單、menu、popup 或 listbox 偶爾被第一個對話窗格遮住的問題。
- 修正開啟 Project 設定時，畫面可能全灰但設定面板沒有顯示的問題。
- 修正 Project 指令及其他動態設定內容增高、縮小、刪除或清空後，設定面板裁切範圍沒有同步的問題。
- 防止舊 dialog 或 popup rect 殘留，避免新舊設定面板或彈出視窗重疊。
- 防止 sidebar WebContents 的原始 main workspace 與設定面板同時顯示。
- 新增集中式 overlay control、shape 與狀態政策。
- 新增安全的 overlay、dialog、popup 與 isolation diagnostics。
- 新增 43 個單元測試及 1 個離線 Electron fixture，合計 44 個自動測試。
- 保持一般對話、Project 對話、Project 新對話、Settings、Search、外部帳號頁面及既有 pane 行為。
- 4 格或 6 格切換對話時可能出現的短暫背景閃爍仍屬已知限制，本版本未處理。

## v4.5.6 — 2026-07-12

- 修正從 Project 建立新對話時，指定活動窗格沒有進入 Project 新對話的問題。
- 使用明確且一次性的 Project 操作意圖，處理非 anchor 控制項產生的路由。
- 防止關閉 Settings 或點擊背景後，活動窗格被誤切換至其他對話。
- 保持一般對話、Project 既有對話、Settings、Search 與外部帳號頁面的既有行為。
- 新增隱私安全的本機 JSONL 診斷日誌與 `npm run diagnostics`。
- 新增路由政策與診斷回歸測試基線，共 32 個自動測試。
- `npm run verify` 現在會同時執行 Repository 基線檢查與自動測試。
- 4 格或 6 格切換對話時可能出現的短暫背景閃爍仍屬已知限制，本版本未處理。

## v4.5.5 — 2026-07-11

- 新增 `Ctrl + Alt + W` 關閉目前活動窗格，支援 `4 → 3`、`3 → 2`、`2 → 1`。
- 1 格與 6 格布局禁止單格關閉；無法關閉時會顯示不搶焦點的非阻塞提示。
- 新增 `Ctrl + Alt + Shift + ←／→`，交換活動窗格與相鄰窗格。
- 窗格交換不重新建立或重新載入 `WebContentsView`，也不中斷背景回答。
- 縮減布局時自動保留目前活動窗格。
- 窗格順序與 URL 會同步保存，重新啟動後維持交換後順序。
- 新增 `AGENTS.md` 與 `npm run verify` 開發安全基線。
- 仍不支援 5 格布局。

## v4.5.4.2 — 2026-07-11

- 將右側窗格切換限制為明確的側欄點擊意圖。
- 忽略官方側欄自行產生的工作區導覽，避免關閉設定後切換成新對話。
- 移除不穩定的背景點擊幾何判斷與計時補丁。
- 修正 Project 名稱包含「設定」或「Settings」時被誤判為官方設定功能。
- 保留設定關閉按鈕、Escape、背景關閉、搜尋與 Project 對話切換功能。

## v4.5.4.1

- 修正設定頁面中原本隱藏的選項文字被強制顯示並重疊的問題。
- 同步修正主程序與 sidebar preload 中兩份重複的透明覆蓋 CSS。
- 設定、搜尋、選單與清單完全沿用 ChatGPT 官方的顯示與隱藏狀態。
- 修正關閉設定或點擊背景後，活動窗格被切換至其他對話或首頁的問題。
- 此修正版不包含關閉窗格或窗格交換功能。

## v4.5.4

- 新增 `Ctrl + Alt + R`，重新整理目前選取的對話窗格。
- 同時監聽全域快捷鍵與目前聚焦頁面的鍵盤事件。
- 重新整理目前窗格時，同步重新整理官方左側功能欄。
- 新增 `Ctrl + Alt + ↑` 與 `Ctrl + Alt + ↓`，選擇垂直相鄰窗格。
- 從多窗格切換為單窗格時，保留目前選取的對話。
- 被保留的對話不會因切換為單窗格而重新載入。
- 若指定方向沒有相鄰窗格，不會跨列或循環選取。

## v4.5.3

- 降低右側窗格對 ChatGPT DOM 的無效重複掃描。
- 降低官方左側欄覆蓋視窗的形狀重算頻率。
- 避免覆蓋視窗形狀與位置未變時重複更新。
- 活動窗格切換時只更新前後兩個窗格的外框。
- 4 與 6 窗格改為分批載入，降低同時初始化造成的瞬間卡頓。
- 保留背景回答生成、完整多窗格功能與硬體加速。
- 延續 v4.5.2 的灰色畫面、右側內部側邊欄與路由修復。
