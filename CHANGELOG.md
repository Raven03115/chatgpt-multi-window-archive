# Changelog

本文件記錄各版本的重要功能、修正與行為變更。

## v4.5.6 — 2026-07-12

- 修正從 Project 建立新對話時，指定活動窗格沒有進入 Project 新對話的問題。
- 使用明確且一次性的 Project 操作意圖，處理非 anchor 控制項產生的路由。
- 防止關閉 Settings 或點擊背景後，活動窗格被誤切換至其他對話。
- 保持一般對話、Project 既有對話、Settings、Search 與外部帳號頁面的既有行為。
- 新增隱私安全的本機 JSONL 診斷日誌與 `npm run diagnostics`。
- 新增路由政策與診斷回歸測試基線，共 32 個自動測試。
- `npm run verify` 現在會同時執行 Repository 基線檢查與自動測試。
- 4 格或 6 格切換對話時可能出現的短暫背景閃爍仍屬已知限制，本版本未處理。

## v4.5.5 — 2026-07-11

- 新增 `Ctrl + Alt + W` 關閉目前活動窗格，支援 `4 → 3`、`3 → 2`、`2 → 1`。
- 1 格與 6 格布局禁止單格關閉；無法關閉時會顯示不搶焦點的非阻塞提示。
- 新增 `Ctrl + Alt + Shift + ←／→`，交換活動窗格與相鄰窗格。
- 窗格交換不重新建立或重新載入 `WebContentsView`，也不中斷背景回答。
- 縮減布局時自動保留目前活動窗格。
- 窗格順序與 URL 會同步保存，重新啟動後維持交換後順序。
- 新增 `AGENTS.md` 與 `npm run verify` 開發安全基線。
- 仍不支援 5 格布局。

## v4.5.4.2 — 2026-07-11

- 將右側窗格切換限制為明確的側欄點擊意圖。
- 忽略官方側欄自行產生的工作區導覽，避免關閉設定後切換成新對話。
- 移除不穩定的背景點擊幾何判斷與計時補丁。
- 修正 Project 名稱包含「設定」或「Settings」時被誤判為官方設定功能。
- 保留設定關閉按鈕、Escape、背景關閉、搜尋與 Project 對話切換功能。

## v4.5.4.1

- 修正設定頁面中原本隱藏的選項文字被強制顯示並重疊的問題。
- 同步修正主程序與 sidebar preload 中兩份重複的透明覆蓋 CSS。
- 設定、搜尋、選單與清單完全沿用 ChatGPT 官方的顯示與隱藏狀態。
- 修正關閉設定或點擊背景後，活動窗格被切換至其他對話或首頁的問題。
- 此修正版不包含關閉窗格或窗格交換功能。

## v4.5.4

- 新增 `Ctrl + Alt + R`，重新整理目前選取的對話窗格。
- 同時監聽全域快捷鍵與目前聚焦頁面的鍵盤事件。
- 重新整理目前窗格時，同步重新整理官方左側功能欄。
- 新增 `Ctrl + Alt + ↑` 與 `Ctrl + Alt + ↓`，選擇垂直相鄰窗格。
- 從多窗格切換為單窗格時，保留目前選取的對話。
- 被保留的對話不會因切換為單窗格而重新載入。
- 若指定方向沒有相鄰窗格，不會跨列或循環選取。

## v4.5.3

- 降低右側窗格對 ChatGPT DOM 的無效重複掃描。
- 降低官方左側欄覆蓋視窗的形狀重算頻率。
- 避免覆蓋視窗形狀與位置未變時重複更新。
- 活動窗格切換時只更新前後兩個窗格的外框。
- 4 與 6 窗格改為分批載入，降低同時初始化造成的瞬間卡頓。
- 保留背景回答生成、完整多窗格功能與硬體加速。
- 延續 v4.5.2 的灰色畫面、右側內部側邊欄與路由修復。
